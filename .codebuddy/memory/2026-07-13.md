# 2026-07-13

## 分析 coros_api_sync.py 字段获取方法

分析了 `run_page/coros_api_sync.py` 各字段来源，对照 `docs/coros-detail-query-plan.md`。
关键结论：
- `distance`: summary.distance ÷100（厘米→米）
- `moving_time`: summary.workoutTime ÷100
- `elapsed_time`: summary.totalTime ÷100，缺失回退时间戳相减
- `start_date_local`: summary.timezone × 15分钟 得 UTC 偏移（32=UTC+8）
- `average_speed`: distance/moving_time（不用 avgSpeed，其单位随运动类型变化）
- `map`: frequencyList gpsLat/gpsLon ÷1e7 → polyline.encode
- `id`: startTimestamp × 10，与 FIT run_id 兼容无重复行

## location_country 字段取值分析
- db.py 内有两处填充：① `update_or_create_activity` 建记录时内联 `reverse_geocode` 兜底（location_country 空或=="China"）；② `backfill_location_country` 事后批量补跑反查。底层都走 `reverse_geocode`（geopy Nominatim）。
- 各 sync 脚本来源不同：coros/codoon/oppo/tulipsport/nike/endomondo 留空交反查；keep 用 `region`（行政区名）；joyrun 用 `city:province`。
- 字段为 String，格式不统一：Nominatim 逗号分隔地址串 / `city:province` / 行政区名三者之一（CLAUDE.md 行55 印证）。

## 统一 location_country 取值方式
- 删除 keep_sync.py（region）与 joyrun_sync.py（city:province）自带的非规范地名赋值，改为 `location_country = ""`，统一交给 `update_or_create_activity` 内联 `reverse_geocode` + `backfill_location_country` 后期反查。
- 注意：`update_or_create_activity` 的「更新分支」不刷新 location_country，已入库的旧 keep/joyrun 非规范记录不会被自动覆盖，需另行处理或依赖手动重跑 backfill。
